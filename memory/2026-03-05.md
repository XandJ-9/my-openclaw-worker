## Rate Limit 优化

**核心原则**：避免重复请求，合并任务，批量处理

**优化策略**：
1. 任务合并：多个小任务合并为一次操作
2. 批量处理：多个网页/文件一起处理
3. 缓存优先：先检查是否有现成数据
4. 延迟重试：遇到 rate limit 时自动等待

**具体场景**：
- 批量读取文件，而不是逐个处理
- 合并指令，减少调用次数
- 等待 1-5 秒后重试，而不是立即重试
