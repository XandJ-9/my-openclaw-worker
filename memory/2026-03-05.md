## Rate Limit 优化

**核心原则**：避免重复请求，合并任务，批量处理

**优化策略**：
1. 任务合并：多个小任务合并为一次操作
2. 批量处理：多个网页/文件一起处理
3. 缓存优先：先检查是否有现成数据
4. 延迟重试：遇到 rate limit 时自动等待

**具体场景**：
- 批量读取文件，而不是逐个处理
- 合并指令，减少调用次数
- 等待 1-5 秒后重试，而不是立即重试

## 模型配置

**全局默认模型**：
- 路径：`agents.defaults.model.primary`
- 当前设置：`zai/glm-4.7-flashx`
- 作用：所有会话（包括 cron）默认使用此模型

**如何设置**：
```bash
openclaw config set agents.defaults.model.primary "zai/glm-4.7-flash"
```

**重要**：
- `agents.defaults.model.primary` 是全局默认模型
- 所有会话（main、isolated、cron）都会使用这个模型
- 如果需要不同会话使用不同模型，需要通过 `agents.list[].model` 或 cron 任务的 `--model` 参数单独指定
